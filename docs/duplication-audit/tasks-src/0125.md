## Required Report Shape

1. Summary: top duplication candidates for this primary file, or `No meaningful duplication found`.
2. Coverage TSV: exactly one row per required symbol.
3. Findings: each real duplication with file/line references, why behavior is duplicated, and differences to preserve.
4. Proposed generalization: minimal helper/module location, or `No refactor recommended`.
5. Risk/tests: behavior risks and tests that would need attention.

Keep findings factual and conservative.
Do not propose broad architecture changes unless the duplication is clearly active and repeated.
